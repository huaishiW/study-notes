# 引论
## 1、操作系统的目标与作用
### 处理机管理
- 用于分配内存和控制处理机
### 存储器管理
- 负责内存的分配与回收，内存保护，地址映射，内存扩充
### I/O设备管理
- 负责I/O设备的分配（回收）与操纵
### 文件管理
- 对文件的存取、共享、保护；具有对文件存储空间的管理，目录管理，文件的读/写管理，文件的共享与保护等功能
## 3、操作系统的基本特征
### 互斥共享方式
- 系统中的某些资源虽然可以提供给多个进程（线程）使用，但应规定在一段时间内，只允许一个进程访问该资源。为此在系统中建立一种机制，以保证多个进程对这类资源的互斥访问
### 同时访问方式
- 系统中还有另一类资源，允许在一段时间内由多个进程”同时“访问。这里的”同时“指的是宏观意义上的同时，在微观上，这些进程对该资源的访问时交替进行的
# 进程的描述与控制
## 2、进程
### 就绪状态
- 指进程已处于准备好运行的状态，即进程已分配到除 CPU 之外的所有必要资源后，只要获得 CPU 便可立即执行。
- **就绪队列**：系统中有很多个这样的处于就绪状态的进程，通常将他们按照一定的策略排成的队列
### 执行状态
- 指进程已经获取 CPU 其程序正在执行的状态
### 阻塞状态
- 指正在执行的进程由于发生某事件在世无法继续执行时的状态，即进程的执行受到
