# 一、处理机调度的层次和调度算法的目标
## 1、调度层次
### （1）高级调度（作业调度）
- 根据某种算法，决定后备队列中的哪几个作业调入内存分配必要的资源，并将它们放入就绪队列
- 主要适用于**多道批处理系统**，**分时系统和实时系统无作业调度**
### （2）中级调度
- 为提高内存利用率和系统吞吐量，将暂时不运行的进程从内存调至外存，适当时机再将其调回内存
- 起到内存扩充的作用
![](操作系统/attachments/Pasted%20image%2020260418192457.png)
### （3）低级调度（进程调度）
根据某种算法，决定就绪队列中的哪个进程应获得处理机，并由分派程序将处理机分配给被选中的进程。**所有的系统都有进程调度**
### （4）调度队列模型
![](操作系统/attachments/Pasted%20image%2020260418192712.png)
![](操作系统/attachments/Pasted%20image%2020260418192731.png)
与上一模型的主要区别：
- 就绪队列的形式
- 设置多个阻塞队

## 2、调度算法的目标
共同目标：
- 资源利用率：
![](操作系统/attachments/Pasted%20image%2020260418193022.png)
- 公平性（避免进程饥饿现象）
- 平衡性（CPU型+I/O型）
- 策略强制执行（安全策略避免死锁）
### （1）批处理系统目标
1. **平均周转时间短**：从作业提交给系统开始，到作业完成为止的时间间隔
   - 周转时间 = 完成时间 - 到达时间
   - 带权周转时间 = 周转时间 / 运行时间
2. **系统吞吐量高**（短作业优先）
3. **处理机利用率高**
>周转时间一般包括：作业在外存后备队列上的等待时间；进程在就绪队列上的等待时间；进程在CPU上的运行时间；等待I／O操作完成的时间
### （2）分时系统目标
- **响应时间快**：从用户提交请求到系统首次响应的时间
- **均衡性**：响应时间应与用户请求服务的复杂性相适应
### （3）实时系统目标
- **截止时间保证**：任务开始执行的最迟时间或必须完成的最迟时间
- **可预测性**：可提高实时性
# 二、作业与作业调度
## 1、批处理系统中的作业
### （1）作业和作业步
- **作业**（Job）:除程序和数据外，配有一份作业说明书
- **作业步**（Job Step）：作业中相互独立又相互关联的加工步骤
### （2）作业控制块（JCB）
作业在系统中存在的标志，保存了系统对作业进行管理和调度所需的全部信息其中包括：
- 作业标识、用户名称、用户账号
- 作业类型（CPU繁忙型、I/O繁忙型、批量型、终端型）
- 作业状态、调度信息（优先级、作业运行时间）
- 资源需求（预计运行时间、要求内存大小等）
- 资源使用情况等
### （3）作业运行的三个阶段和三种状态
1. **收容阶段**（后备状态）：作业在外存后备队列上等待
2. **运行阶段**（运行状态）：被作业调度选中，分配资源、建立进程并放入就绪队列
3. **完成阶段**（完成状态）：进程释放资源
## 2、调度算法
- 调度算法性能的衡量：平均周转时间、平均带权周转时间
	- 周转时间 = 完成时间 - 到达时间
	- 带权周转时间 = 周转时间 / 运行时间
### （1）先来先服务（FCFS）
- 既可用于作业调度又可用于进程调度
- 不单独作为一个调度算法用于实际的操作系统中，但常常和其它算法配合使用
- 特点：
	- 利于长作业（进程），而不利于短作业（进程）
	- 利于CPU繁忙型作业（进程），而不利于I／O型繁忙作业（进程）
	- 作业平均等待时间长
	- 系统吞吐量不高
### （2）短作业优先（SJF）/ 短进程优先（SPF）
- **SJF**：从后备队列中选择**一个或多个**估计运行时间最短的作业调入内存
- **SPF**：从就绪队列中选择**一个**估计运行时间最短的进程
- **优点**：能有效降低作业平均等待时间，提高系统吞吐量
- **缺点**：对长作业不利，可能出现饥饿现象；未考虑作业的紧迫程度

### 3.4.3 优先级调度算法
- 基于作业的紧迫程度由外部赋予优先级，越紧迫的作业优先级越高。
- 可用于作业调度，也可用于进程调度
### 3.4.4 高响应比优先调度算法（动态优先权）
响应比 = (等待时间 + 服务时间) / 服务时间
**算法特点**：
1. 等待时间相同时，服务时间愈短，优先权愈高，有利于短作业
2. 服务时间相同时，等待时间愈长，优先权愈高（类似FCFS）
3. 长作业的优先级可随等待时间增加而提高，避免饥饿
# 三、进程调度
## 1、进程调度的任务、机制、方式
### （1）任务
- 保存处理机的现场信息
- 按某种算法选取进程
- 把处理器分配给进程
### （2）机制
- 排队器
- 分派器
- 上下文切换器
![](操作系统/attachments/Pasted%20image%2020260418195922.png)
### （3）调度方式
#### 非抢占方式
一旦处理机分配给某进程则一直让它运行，直至该进程运行完毕或阻塞时才让出处理机
引起进程调度的因素：
- 正在执行的进程运行完毕，或因发生某事件而无法继续运行
- 进程提出 I/O 请求，暂停执行
- 进程通信或同步中执行了某种原语操作
#### 可抢占方式
允许系统调度程序终止某个进程的运行，将处理机重新分配给其它进程。
**抢占原则**：
1. 时间片原则
2. 优先权原则
3. 短作业优先原则
## 2、轮转调度算法
### （1）基本原理
- 将CPU时间划分成大小相同的时间片，并将所有就绪进程**按到达的先后次序排成队列**，每次调度都是**从队首选择一个进程**使其运行
- 每次调度只运行一个时间片，若在本时间片中进程运行结束，则调度下一个队首进程
- 若时间片耗尽时当前进程未结束，则终止该进程运行，将其**插入到就绪队列的队尾**，再从**队首选择下一个进程**使其运行
- 可以保证就绪队列中的所有进程，在一个确定的时间段内，都能获得一次CPU执行
### （2）进程切换时机
- 在时间片轮转调度算法中，应在何时进行进程的切换，可分为两种情况：
	- 若一个时间片尚未用完，正在运行的进程便已经完成，就立即激活调度程序，调度就绪队列中队首的进程运行，并启动一个新的时间片
	- 在一个时间片用完时，计时器中断处理程序被激活。如果进程尚未运行完毕，调度程序将把它送往就绪队列的末尾
### （3）时间片大小
- 在轮转算法中，时间片的大小对系统性能有很大的影响
## 3、优先级调度算法
### （1）算法类型
- 非抢占式优先权算法：系统一旦把处理机分配给就绪队列中优先权最高的进程后，该进程便一直执行下去，直至完成或因发生某事件使该进程放弃处理机，此时，系统方可再将处理机重新分配给另一优先权最高的进程
`主要用于批处理系统中，也可用于某些对实时性要求不严格的实时系统中`
- 抢占式优先权算法：系统同样是把处理机分配给优先权最高的进程，使之执行。**但在其执行期间，只要出现了另一个优先权更高的进程，进程调度程序就立即停止当前进程(原优先权最高的进程)的执行，重新将处理机分配给新到的优先权最高的进程**
`能更好地满足紧迫作业的要求，故常用于对实时性要求比较严格的系统中`
### （2）优先级类型
确定进程优先级的依据有三个方面：进程类型、进程对资源的需求、 用户要求
- **静态优先级**：在创建进程时确定的，且在进程的整个运行期间保持不变
`简单易行、开销小优先级低进程会出现饥饿现象`
- **动态优先级**：在创建进程时所赋予的优先级，可以随进程的推进或随其等待时间的增加而改变
## 4、多级反馈队列调度算法
只用于进程调度、思想如下：
- 设置多个就绪队列，不同队列具有不同优先级，第一个队列优先级最高，以后次之
- 给不同队列分配不同大小的时间片，第一个队列最小，以后次之（皆为前者的二倍），有的系统最后一级队列不划分时间片
- 新进程进入第一级队列的队尾
- 每一级队列均按FCFS算法调度
- 执行的进程在规定的时间片内未执行完毕，则进入下一级队列的队尾
- 仅当前一级队列为空时才调度下一级队列中的进程
- 算法采用抢占式调度策略
![](操作系统/attachments/Pasted%20image%2020260418203528.png)
- 算法性能：
	1.终端型作业用户
	2.短批处理作业用户
	3.长批处理作业用户
# 四、实时调度
## 1、实现实时调度的基本条件
### （1）提供必要的信息
- 就绪时间、开始截止时间和完成截止时间、处理时间、资源要求、优先级
### （2）系统处理能力强
假定系统中有 m 个周期性硬实时任务，它们的处理时间为 $C_i$，周期时间为 $P_i$，则在单处理机情况下，必须满足下面的限制条件系统才是可调度的
$$\sum_{i=1}^{m} \frac{C_i}{T_i} \leq 1$$
### （3）采用抢占式调度机制
- 当一个优先权更高的任务到达时，允许抢占当前任务处理机，令高优先权任务立即投入运行，这样便可满足该硬实时任务对截止时间的要求
- 对于一些小的实时系统，如果能预知任务的开始截止时间，则对实时任务的调度可采用非抢占调度机制，以简化调度程序和任务调度时所花费的系统开销
### （4）具有快速切换机制
- 对外部中断的快速响应能力
- 快速的任务分派能力

## 3、常用的几种实时调度算法
### （1）最早截止时间优先算法（EDF）
- 根据任务的**截止时间**来确定任务的优先级。截止时间愈早，其优先级愈高
- 实时任务就绪队列按各任务的截止时间的早晚排序
- 可用于抢占式调度，也可用于非抢占式调度
### （2）最低松弛度优先调度（LLF）
- 根据**任务紧急（或松弛）的程度**来确定优先级，任务紧急程度愈高，优先级愈高。
- 实现该算法要求系统中有一个按松弛度排序的实时任务就绪队列，松弛度最低的任务排在队列的最前面
- 主要用于可抢占式调度方式中
# 五、死锁（Deadlock）

### 3.7.1 资源分类
- **可抢占性资源**：进程在占有时可以被强行夺走
- **非抢占性资源**：系统将这类资源分配给进程后，不能强行收回

### 3.7.2 死锁的必要条件
1. **互斥条件**：资源一次只能被一个进程使用
2. **请求和保持条件**：进程在保持资源的同时请求新资源
3. **不可抢占条件**：已分配资源不能被强行夺走
4. **循环等待条件**：形成进程-资源的循环链

### 3.7.3 死锁的预防
通过设置限制条件，破坏四个必要条件中的一个：

| 条件 | 预防方法 |
|------|----------|
| 互斥条件 | 一般不预防（有些资源必须互斥访问） |
| 请求和保持条件 | 一次性申请整个运行过程中的全部资源 |
| 不可抢占条件 | 当资源请求不满足时，释放已占资源 |
| 循环等待条件 | 系统将所有资源按类型线性排队，赋予不同序号 |

**优点**：实现简单

### 3.7.4 死锁的避免
#### 安全状态
系统能按某种进程推进顺序(P1, P2, …, Pn)，为每个进程分配所需资源，直至满足最大需求，使每个进程都可顺利完成。

**安全序列**：使系统处于安全状态的进程执行顺序

#### 银行家算法（Banker's Algorithm）
**数据结构**：
- `Available[m]`：每类可利用资源数目
- `Max[n,m]`：每个进程对每类资源的最大需求
- `Allocation[n,m]`：每类资源已分配给每个进程的数量
- `Need[n,m]`：每个进程尚需的各类资源数

**算法步骤**：
1. 如果 Request_i ≤ Need：继续；否则出错
2. 如果 Request_i ≤ Available：继续；否则进程等待
3. 系统试探分配，修改数据结构
4. 系统执行安全性检查，若安全则真正分配

**例**：P4请求资源Request4(3,3,0)
- 检查：Request4(3,3,0) ≤ Need4(4,3,1) ✓
- 检查：Request4(3,3,0) > Available(2,3,0)，让P4等待

### 3.7.5 死锁的检测
#### 资源分配图简化法
1. 消去资源分配图中申请边和分配边
2. 如果能简化所有进程结点到孤立结点，则图是完全可简化的
3. 否则系统会发生死锁

#### 银行家算法检测死锁
```
Work = Available
L = {Li | Allocationi = 0 且 Requesti = 0}

while (存在满足Requesti ≤ Work的进程) {
    Work = Work + Allocationi
    将其记入L表
}

if (L中包含所有进程) → 无死锁
else → 发生死锁
```

### 3.7.6 死锁的解除
1. **撤消死锁进程**：按优先级、执行时间、资源等原则逐个撤消
2. **剥夺资源**：剥夺死锁进程占用的资源

## 3.8 补充概念

### 周转时间
- 周转时间 = 完成时间 - 到达时间
- 带权周转时间 = 周转时间 / 运行时间

### 饥饿 vs 死锁
- **饥饿**：进程长期得不到所需资源（可预防）
- **死锁**：进程间相互等待对方资源（四个必要条件）

### 调度指标
- CPU利用率
- 吞吐量
- 周转时间
- 响应时间
