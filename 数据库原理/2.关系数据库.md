# 一、关系数据结构及形式化定义
## 1. 关系的数学定义
### （1）域（Domain）
- 具有相同数据类型值的集合
### （2）笛卡儿积（Cartesian Product）
- **定义**：给定一组域 D1, D2, ..., Dn，域可以部分、全部、完全不相同。笛卡儿积为：
  ```
  D1 × D2 × ... × Dn = {(d1,d2,...,dn) | di ∈ Di, i=1,2,...,n}
  ```
- **(d1,d2,...,dn)** 称为元组
- 若Di为有限集，其基数为mi，则笛卡儿积的基数为 m1×m2×...×mn
### （3）关系（Relation）
- D1×D2×...×Dn的子集叫作在域D1, D2, ..., Dn上的关系
- 表示为：R(D1, D2, ..., Dn)
  - R：关系名
  - n：关系的目或度（Degree）
**关系的性质**：
- **列可交换**：关系满足交换律，笛卡尔积不满足
- **有限子集**：关系是笛卡尔积的有限子集
## 3. 码的概念（属性）
- **定义**：关系中不同列可以对应相同的域，为了加以区分，必须对每列起一个名字

| 概念       | 说明                           | 性质      |
| :------- | :--------------------------- | ------- |
| **超码**   | 关系中唯一标识元组的属性组                | 唯一性     |
| **候选码**  | 关系中某一超码，去掉其中任一属性后，均不再为超码     | 唯一性、最小性 |
| **主码**   | 若一个关系有多个候选码，则选定其中一个为主码       |         |
| **主属性**  | 候选码的各个属性                     |         |
| **非主属性** | 不包含在任何候选码中的属性                |         |
| **全码**   | 在极端情况下，关系模式的所有属性组是这个关系模式的候选码 |         |

## 4. 三类关系

| 类型 | 说明 |
|------|------|
| **基本表** | 实际存在的表、实际存储数据的逻辑表示 |
| **查询表** | 结果表或查询中临时生成的表，冗余性和一次性，是从基本表中派生出来的 |
| **视图** | 虚表，不对应实际存储的数据，依附于基本表 |

## 5. 关系基本性质
1. 同一属性的数据是同质的，同一列中的分量来自同一个域，不同的列可以出自同一个域
2. 同一关系的属性名具有==不能重复==性
3. 列的位置==与次序无关==
4. **元组无冗余性**：任意二个元组不能完全相同
5. 元组的位置==与顺序无关==
6. 每一个分量（属性）必须是==不可分==的数据项
## 6. 关系模式（Relation Schema）
**是对关系的描述**，包括：
- 元组集合的结构
	- 属性构成
	- 属性来自的域
	- 属性与域之间的映象关系
- 元组语义以及完整性约束条件
	- 属性间的数据依赖关系集合
### （1）形式化定义：R(U, D, dom, F)
- R：关系模式名
- U：属性集
- D：U中属性的值所来自的域
- Dom：属性向域的映射集合
- F：属性间的依赖关系
### （2）关系模式与关系的区别

| 概念 | 特点 |
|------|------|
| 关系模式 | 对关系的描述，静态的、稳定的 |
| 关系 | 关系模式在某一时刻的状态或内容，动态的、随时间不断变化的 |
| 关系数据库模式 | 对关系数据库的描述，包括若干域的定义和若干关系模式 |
| 关系数据库 | 这些关系模式在某一时刻对应的关系的集合 |

# 二、关系操作
## 1. 关系操作概述
- 关系操作包括**数据查询**和**数据维护**
- 操作的对象和结果都是集合
- 关系操作语言是**高度非过程化语言**
## 2. 关系操作语言分类

| 类型 | 说明 | 典型代表 |
|------|------|----------|
| **关系代数语言** | 用对关系的运算来表达查询要求 | ISBL |
| **元组关系演算语言** | 用谓词来表达查询要求，谓词变元的基本对象是元组变量 | ALPHA, QUEL |
| **域关系演算语言** | 谓词变元的基本对象是域变量 | QBE |
| **二者结合** | 具有关系代数和关系演算双重特点的语言 | SQL |

# 三、关系的完整性
## 1. 三类完整性约束

| 约束类型                                | 说明                        |
| ----------------------------------- | ------------------------- |
| **实体完整性(Entity Integrity)**         | 若属性A是基本关系R的主码属性，则属性A不能取空值 |
| **参照完整性(Referential Integrity)**    | 外码要么取空值，要么等于被参照关系中已存在的主码值 |
| **用户定义完整性(User-defined Integrity)** | 针对某一具体关系数据库的约束条件，反映语义要求   |
- 实体完整性与参照完整性是关系的两个不变性，DBMS自动支持
## 2. 实体完整性
- **规则**：若属性A是基本关系R的主码属性，则属性A不能取空值。
- **作用**：
	- 能够保证实体的唯一性
	- 能够保证实体的可区分性
`注意：违反了实体完整性规则，在现实世界中是没有意义的`
## 3. 参照完整性
### （1）定义
- 在关系模型中实体及实体间的联系都是用关系来描述的，因此可能存在着关系与关系间的引用
`注：表示关系之间联系的属性可以为空，否则必须和另一个关系的码相等；该属性可以不唯一；属性名可以不同`
### （2）外码（Foreign Key）
- **定义**：设F是基本关系R的一个或一组属性，但不是关系R的码。如果F与基本关系S的主码Ks相对应
- 基本关系R称为**参照关系**
- 基本关系S称为**被参照关系**或**目标关系**
说明：
	- 关系 R 和 S 不一定是不同的关系
	- 目标关系 S 的主码 Ks 和参照关系的外码F必须定义在同一个（或一组）域上
	- 外码并不一定要与相应的主码同名
	- 当外码与相应的主码属于不同关系时，往往取相同的名字，以便于识别

### （3）参照完整性规则
- 外码可以取空值
- 外码也可以取被参照关系中已存在的主码值
- 目标关系的主码和外码必须定义在同一个（或一组）域上
- 外码并不一定要与相应的主码同名

## 4. 用户定义的完整性
- 针对某一具体关系数据库的约束条件，反映某一具体应用所涉及的语义要求。
- 关系模型应提供定义和检验这类完整性的机制，以便用统一的系统的方法处理它们，而==不要由应用程序承担这一功能==
# 四、关系代数
## 1. 关系代数概述
**定义**：一种抽象的查询语言，用对关系的运算来表达查询。
**运算三要素**：
- 运算对象：关系
- 运算结果：关系
- 运算符：四类
## 2. 运算符分类

| 运算符类别 | 运算符 | 说明 |
|------------|--------|------|
| **集合运算符** | ∪ (并), - (差), ∩ (交), × (广义笛卡尔积) | 将关系看成元组的集合，运算是从关系的"水平"方向即行的角度来进行 |
| **专门的关系运算符** | σ (选择), π (投影), ⋈ (连接), ÷ (除) | 不仅涉及行而且涉及列 |
| **比较运算符** | ＞, ≥, ＜, ≤, ＝, <> | 辅助专门的关系运算符进行操作 |
| **逻辑运算符** | ┐ (非), ∧ (与), ∨ (或) | 辅助专门的关系运算符进行操作 |

## 3. 传统集合运算
### （1）并运算 (∪)
- R 和 S 具有相同的目 n（即两个关系都有n个属性），相应的属性取自同一个域
- R∪S 仍为 n 目关系，由属于 R 或属于 S 的元组组成
- **公式**：R∪S = { t | t ∈ R ∨ t ∈S }
### （2） 差运算 (-)
- R 和 S 具有相同的目 n（即两个关系都有n个属性），相应的属性取自同一个域
- R - S 仍为 n 目关系，由属于 R 而不属于 S 的元组组成
- **公式**：R - S = { t | t∈R ∧ t∉S }
### （3）交运算 (∩)
- R 和 S 具有相同的目 n（即两个关系都有n个属性），相应的属性取自同一个域
- R ∩ S 仍为 n 目关系，由既属于 R 又属于 S 的元组组成
- **公式**：R ∩ S = { t | t ∈ R ∧ t ∈S } = R - (R-S)
### （3）广义笛卡尔积 (×)
- R: n目关系，k1个元组
- S: m目关系，k2个元组
- R×S: 列( n + m )列元组的集合，元组的前 n 列是 R 的一个元组，后 m 列是 S 的一个元组
- 行：k1×k2个元组
- **公式**：R×S = {tr ts | tr ∈R ∧ ts∈S }
## 4. 专门的关系运算
### （1）选择（Selection）σ
- **定义**：在关系R中选择满足给定条件的诸元组。
- **公式**：σF(R) = {t | t∈R ∧ F(t) = '真'}
**F：选择条件**，基本形式为：
```
[┐( ] X1θY1 [ )][∧ [┐( ] X2θY2 [ )]]...
```
- θ：比较运算符（＞，≥，＜，≤，＝或<>）
- X1, Y1等：属性名、常量、简单函数
- ┐：逻辑运算符（∧或∨）
### （2）投影（Projection）π
- **定义**：从 R 中选择出若干属性列组成新的关系。
- **公式**：πA(R) = { t \[ A ] | t ∈R }
`注意：投影之后不仅取消了原关系中的某些列，而且还可能取消某些元组（避免重复行）`
### （3）连接（Join）⋈
- **定义**：从两个关系的笛卡尔积中选取属性间满足一定条件的元组。
- **公式**：
```
R ⋈ S = { tr ts | tr∈R ∧ ts∈S ∧ tr[A] θ ts[B] }
```
- **两类常用连接**：
	- 等值连接（equijoin）： θ为“＝”的连接运算称为等值连接
		- **含义**：从关系R与S的广义笛卡尔积中选取A、B属性值相等的那些元组
	- 自然连接（Natural join）：自然连接是一种特殊的等值连接
		- **含义**：两个关系中进行比较的分量必须是相同的属性组，在结果中把重复的属性列去掉（R和S具有相同的属性组B）

**外连接**：在自然连接中把舍弃的元组也保存在结果关系中，其它属性上填空值(Null)
- **左连接**：只保留左边关系要舍去的元组
- **右连接**：只保留右边关系要舍去的元组
### （4）除（Division）÷
- **定义**：给定关系 R(X, Y) 和 S(Y, Z)，R÷S 得到一个新的关系 P(X)，P 是 R 中满足下列条件的元组在 X 属性列上的投影：元组在 X 上分量值 x 的象集 Yx 包含 S 在 Y 上投影的集合
- **公式**：R÷S = { tr \[X] | tr ∈ R ∧ πY(S) ⊆ Yx }
- **注意**：除操作是同时从行和列角度进行运算
## 6. 查询步骤
1. 确定有几个属性
2. 根据属性确定涉及几个关系（表），先做多表连接，自然连接注意判断是否有相同属性组
3. 区分选择与投影操作
   - 查询的主题如果是表的若干个属性，用投影
4. 区分差和除操作
   - "没有"，采用差操作，具有否定语义
   - 对于涉及"全部"、"所有"、"至少一个"之类的查询，通常可以考虑除操作
